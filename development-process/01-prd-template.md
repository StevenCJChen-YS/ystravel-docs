# PRD 模板

> 複製這份模板到 `docs/features/<feature-slug>/prd.md`,填空即可。**這份文件由 Steven 主導決定,AI 負責草擬第一版讓 Steven 修改**,不要求 Steven 從零開始寫。

```markdown
# PRD: <功能名稱>

| 項目 | 內容 |
|---|---|
| 功能 slug | <feature-slug> |
| 狀態 | 草稿 / 討論中 / 已核准 / 開發中 / 已上線 |
| 負責決策 | Steven |
| 建立日期 | YYYY-MM-DD |
| 最後更新 | YYYY-MM-DD |

## 1. 一句話摘要

（這個功能是什麼,一句話講完）

## 2. 背景與動機

（為什麼現在需要這個功能?是什麼問題觸發的?不做會怎樣?）

## 3. 目標使用者

（誰會用?內部哪個角色?外部客戶?預估使用頻率?）

## 4. User Story

作為 [角色]
我希望 [能力]
以便 [價值]

## 5. Acceptance Criteria(驗收標準)

- [ ] ...
- [ ] ...
- [ ] ...

> 每一條都要是 Steven 自己能夠實際操作畫面驗證的東西,不要寫成技術實作細節。

## 6. UI / UX 重點

（這塊由 Steven 主導,AI 只是記錄:需要哪些畫面、狀態、互動,列點即可,細節可以之後邊做邊調）

- 需要的畫面 / 狀態:

## 8. 不做的事(Out of Scope)

（明確寫出這次不做什麼,避免範圍蔓延）

## 9. 待釐清問題

- [ ] ...

## 10. 決策紀錄

| 日期 | 決策 | 決定人 |
|---|---|---|
| | | Steven |
```

## 使用方式

1. 跟 AI 描述功能的白話需求(可以很口語,不用先想清楚細節)。
2. AI 依這份模板草擬第一版 PRD。
3. Steven 逐段確認、修改第 3-9 節,第 7 節資安欄位不可省略。
4. 確認後,狀態改為「已核准」,才進入 [02-sa-template.md](./02-sa-template.md)。
